# 数字披萨工厂

一个像素风点击与挂机小游戏。玩家点击披萨核心获得字节，购买虚拟产线提高自动产出，抽取食材卡牌，在厨房组合配方披萨，并通过图鉴、成就、技能树和转生系统形成长期成长目标。

项目基于 Vue 3 + Vite 构建，玩法参考了成熟点击类、挂机类游戏中常见的成长循环：点击反馈、自动产出、抽卡收集、烹饪合成、图鉴目标、离线收益和技能研究。

演示地址：[https://fxxz505.github.io/pizza-factory/](https://fxxz505.github.io/pizza-factory/)

## 在线玩法概览

- 点击披萨核心获得字节资源。
- 使用字节升级点击强度、暴击几率和暴击倍率。
- 购买虚拟产线，获得持续自动产出。
- 抽取随机原料卡，稀有度会影响收集价值和烹调时间。
- 在厨房中组合食材，烘焙披萨并发现配方。
- 通过图鉴查看已发现配方、成就和收集进度。
- 首次发现新披萨会按披萨星级获得技能点，重复配方不会重复加点。
- 在独立技能树页面中加点，强化点击、挂机、抽卡、烹饪、离线收益和转生成长。

## 核心功能

### 点击工厂

工厂页是主要资源入口。点击披萨核心会触发像素碎屑、浮动数字、轻量震动和音效反馈。PC 端采用双列布局，将点击区域、数据面板、挂机产线、运行监控和技能入口拆成清晰模块，避免所有功能挤在同一屏幕区域。

### 挂机产线

玩家可以购买不同产线节点，让系统持续产生字节。产线收益会受到产线数量、技能加成、转生倍率、离线结算效率等因素影响。

### 随机原料抽取

抽卡系统提供单抽和十连抽，并带有最近抽取记录、保底进度、抽取结果展示和成本成长。结果展示保持横向卡片布局，减少十连抽时的跳动和错位。

### 厨房烹饪

玩家可以把抽到的原料放入烹饪槽，组合成不同披萨配方。烹调时间会根据食材稀有度变化，稀有食材通常需要更长的等待时间，也带来更明确的期待感。

### 图鉴与成就

图鉴记录已发现的披萨配方和收集进度，成就系统为点击、抽卡、烹饪、离线收益等行为提供长期目标。

### 技能树与转生

技能树使用披萨星级转化的技能点进行升级。转生系统让长期游玩获得永久加成，使玩家在图鉴收集、技能成长和产能提升之间形成循环。

## 项目结构

```text
pizza-factory-vue/
├── public/                 # 静态资源
├── scripts/                # 测试脚本
├── src/
│   ├── components/         # Vue 页面组件与 UI 模块
│   ├── composables/        # 游戏状态、经济公式、音效、抽卡、烹饪等逻辑
│   ├── data/               # 食材、配方、产线、成就、常量配置

│   ├── App.vue             # 应用入口布局
│   ├── main.js             # Vue 挂载入口
│   └── style.css           # 全局样式、动效和响应式布局
├── index.html
├── package.json
└── vite.config.js
```

## 构建与部署

执行生产构建：

```bash
npm run build
```

构建产物会输出到 `dist/` 目录。仓库已配置 GitHub Pages 工作流，推送到 `main` 分支后会自动构建并发布到：

[https://fxxz505.github.io/pizza-factory/](https://fxxz505.github.io/pizza-factory/)

`vite.config.js` 在生产环境中使用 `/pizza-factory/` 作为 `base`，用于匹配 GitHub Pages 的仓库路径。

## 设计方向

项目视觉采用 8-bit / 像素游戏风格，强调明确的点击反馈、可读的数据模块和轻量的挂机成长。主舞台点击使用更轻的局部反馈，避免频繁点击时模块抖动过强；升级、抽卡、烹饪完成等关键行为保留全屏震动，让奖励时刻更有存在感。
