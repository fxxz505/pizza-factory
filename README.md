# 数字披萨工厂

一个像素风点击与挂机小游戏。玩家通过点击披萨核心获得字节，购买虚拟产线进行自动生产，抽取不同稀有度的食材，再把食材组合成配方披萨并点亮图鉴。

项目基于 Vue 3 + Vite 构建，玩法参考了成熟点击类、挂机类游戏中常见的成长循环：点击反馈、自动产出、抽卡收集、烹饪合成、图鉴目标、离线收益和技能研究。

## 在线玩法概览

- 点击披萨核心获得字节资源。
- 使用字节升级点击强度、暴击几率和暴击倍率。
- 购买虚拟产线，获得持续自动产出。
- 抽取随机原料卡，稀有度会影响收集价值和烹调时间。
- 在厨房中组合食材，烘焙披萨并发现配方。
- 通过图鉴查看已发现配方、成就和收集进度。
- 研究标准技能系统，强化点击、挂机、抽卡和离线收益。

## 核心功能

### 点击工厂

工厂页是主要资源入口。点击披萨核心会触发像素碎屑、浮动数字、震动反馈和音效反馈。PC 端采用双列布局，将点击区域、数据面板、挂机产线、运行监控和技能研究拆成清晰模块，避免所有功能挤在同一屏幕区域。

### 挂机产线

玩家可以购买不同产线节点，让系统持续产生字节。产线收益会受到收藏加成、技能研究、临时事件和过载倍率影响。离线后再次进入游戏时，也会根据挂机产出进行离线结算。

### 随机原料抽取

抽卡系统提供单抽和十连抽，并带有最近抽取记录、保底进度、抽取结果展示和成本成长。技能研究可以降低抽卡成本，让中后期收集更顺滑。

### 厨房烹饪

玩家可以把抽到的原料放入烹饪槽，组合成不同披萨配方。烹调时间会根据食材稀有度变化，稀有食材通常需要更长的等待时间，也带来更明确的期待感。

### 图鉴与成就

图鉴记录已发现的披萨配方和收集进度，成就系统为点击、抽卡、烹饪、离线收益等行为提供长期目标。

### 标准版技能系统

当前技能研究包含四条基础成长方向：

| 技能 | 分支 | 效果 |
| --- | --- | --- |
| 脉冲手套 | 点击核心 | 提升点击收益 |
| 流水线校准 | 挂机产线 | 提升全部挂机产出 |
| 原料扫描仪 | 抽卡烹饪 | 降低抽卡成本 |
| 保温烤箱 | 挂机产线 | 提升离线结算效率 |

## 项目结构

```text
pizza-factory-vue/
├── public/                 # 静态资源
├── scripts/                # 数据生成与测试脚本
├── src/
│   ├── components/         # Vue 页面组件与 UI 模块
│   ├── composables/        # 游戏状态、经济公式、音效、抽卡、烹饪等逻辑
│   ├── data/               # 食材、配方、产线、成就、常量配置
│   ├── assets/             # 项目图片资源
│   ├── App.vue             # 应用入口布局
│   ├── main.js             # Vue 挂载入口
│   └── style.css           # 全局样式与响应式布局
├── index.html
├── package.json
└── vite.config.js
```

## 构建与部署

执行生产构建：

```bash
npm run build
```

构建产物会输出到 `dist/` 目录。该目录可以部署到 GitHub Pages、Vercel、Netlify 或任何静态站点托管服务。

如果部署到 GitHub Pages，并且仓库不是部署在根域名下，可能需要根据仓库路径调整 Vite 的 `base` 配置。

## 设计方向

项目视觉采用 8-bit / 像素游戏风格，强调明确的点击反馈和轻量的挂机成长。PC 端重点解决信息密度与操作空间问题，通过模块化页面、双列布局和紧凑数据卡片提升可读性；移动端保持更直接的单列操作体验。

## 后续可扩展方向

- 增加更多技能分支，例如暴击流、抽卡流、厨房流、图鉴流。
- 增加可重置的转生系统，让长期游玩有更强目标。
- 增加任务系统和每日目标，提升回访动力。
- 增加更多像素动画资源，并接入 Sprite Sheet。
- 将存档从 localStorage 扩展为云存档或账号系统。
- 为关键模块增加更多单元测试和端到端测试。
